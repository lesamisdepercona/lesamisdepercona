MyDumper 0.10.5 est maintenant disponible

By Francis

September 2, 2021 - 2 minutes read - 295 words

La nouvelle version MyDumper 0.10.5 , qui inclut de nombreuses nouvelles fonctionnalités et corrections de bugs, est désormais disponible. Vous pouvez télécharger le code ici

Pour cette version, nous nous sommes concentrés sur la résolution d’anciens problèmes et sur le test d’anciennes demandes d’extraction pour obtenir un code de meilleure qualité. Sur les versions 0.10.1, 0.10.3 et 0.10.5, nous avons publié les packages compilés avec les bibliothèques MySQL 5.7, mais à partir de maintenant, nous compilons également avec les bibliothèques MySQL 8 à guise de test, et non pas de publication, car nous pensons que beaucoup des membres de la communauté commenceront à compiler avec la dernière version, et nous devrions être prêts.

Nouvelles fonctionnalités :

  • Obfuscation du mot de passe #312  
  • Utilisation du paramètre dynamique pour SET NAMES #154 
  • Refactoriser le logging et activer –logfile dans myloader #233 
  • Ajouter l’option de mode purge (NONE/TRUNCATE/DROP/DELETE) pour décider quel est le mode préférable #91 #25  
  • Éviter l’envoi de COMMIT lorsque commit_count est égal à 1 #234 
  • Vérifier si le répertoire existe #294 

Corrections de bugs :

  • Ajout du support de construction homebrew #313  
  • Suppression de la dépendance MyISAM dans les tables temporaires pour VIEWS #261 
  • Correction des avertissements dans la génération de document sphinx #287 
  • Correction d’une boucle sans fin lorsque la liste de processus ne pouvait pas être vérifiée #295 
  • Correction des problèmes lorsque daemon est utilisé sur glist #317 

Documentation :

  • Correction des dépendances des paquets ubuntu / debian #310 
  • Fournir de meilleures instructions de construction CentOS 7 #232 
  • utilisation du fichier –defaults-file pour la section mydumper et le client #306 
  • Ajoutée la documentation INSERT IGNORE #195 

Refactorisation :

  • Ajout de la fonction new_table_job #315 
  • Ajout de IF NOT EXISTS à SHOW CREATE DATABASE #314 
  • Mettre à jour FindMySQL.cmake #149
